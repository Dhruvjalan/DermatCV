## 4. Methodology and Architecture

### Wellness Scan

![Wellness Scanner](test_imgs\Wellness_Scanner.png)

The DermatCV platform implements an enterprise-grade, multi-stage computer vision and deep learning inference pipeline engineered to classify anatomical entry-points, identify specific dermal conditions, localize localized regions of analytical interest via targeted back-activation hooks, and formulate semantic clinical interventions utilizing a localized Retrieval-Augmented Generation (RAG) structure.

### Grab-CAM-based explainability for skin patch analysis. 

To guarantee machine learning translatability, the model uses visual backpropagation weights through Gradient-weighted Class Activation Mapping (GradCAM) on the custom DiseaseClassifierWithGradCAM network to find exactly where the vision layer is paying attention.

* Inference Convolutional Layout: Dermal regions of interest (**ROI**) are passed through three convolutional extraction zones (conv1 through conv3) down-sampling layers to capture structural features

* Gradient Backpropagation Hooks: During execution forward passes, runtime hooks continuously monitor feature maps generated by the terminal convolutional layer (conv3). Upon calculating a target forward prediction, the target class index scalar score **y^c** undergoes a graph-retained backward propagation pass.

* Patch Grid Target Profiling: The system passes the raw output matrix through an activation rectifier (**ReLU**) to extract positive feature importance maps. The activation output map is scaled linearly using bilinear interpolation up to the native **224 x 224** space. This upscaled attention map is split into an **N x N** tracking grid, which isolates the top **P** highly active analytical patches for downstream structural evaluation.


### Autoencoder based Disease Detection

Rather than evaluating large structural frames, high-density anomalies highlighted within the prime localized GradCAM patch are isolated, normalized, downsampled, and evaluated inside an unsupervised bottleneck structure (NormalAutoencoder).

* Latent Feature Projection: The isolated regional patch is interpolated down to a dense **56 x 56** input bounding matrix and passed through stride-2 down-convolutions to extract an intensely compressed semantic representation layer. The layer passes into a flat structural bottleneck to produce a localized latent matrix vector array **v** comprising exactly 8 parameters:

**v = Normalize(W_{fc} . Flatten(Encoder(x)), p=2)**

* Biometric Metrics Derivation: Instead of static linear scoring assumptions, index values are extracted through deterministic algorithmic modulos of the normalized latent dimensions. In this architecture, individual indices are calculated sequentially: vector dimension **v_0** yields the Stress Index, **v_1** maps Surface Fatigue, and **v_2** dictates the baseline Hydration Level. This strategy enables the network to extract continuous wellness metrics straight from latent spatial relationships.



### CNN Based Body Region Classification

The tracking pipeline begins by passing the full image matrix into a custom shallow convolutional neural network (BodyPartClassifier) to verify and register target anatomical positioning across 10 possible channels (such as Face, Skin Hand, Forehead, or Neck).

* Feature Extraction Grid: The architecture processes input arrays shaped **(B, 3, 224, 224)** through a sequential pair of 2D convolutional layers using a **3 x 3 spatial filter stride**, paired directly with inline rectified linear activation units (**ReLU**) and downsampled via a factor-2 MaxPool kernel down to a **56 x 56** feature grid map.

* **Classifier Projection:** The downsampled feature maps are flattened into a single feature vector of size: **32 x 56 x 56 = 100,352  dimensions**

  This vector passes into a hidden fully connected linear projection layer containing **128** hidden nodes, a regularizing Dropout layer set at **p = 0.3** to limit structural overfitting during execution, and a terminal linear classification exit layer mapping logits directly to target body parts.

### Modular FastAPI backend with MongoDB Atlas for user and scan data management.

The system is built on an asynchronous, decouple-ready architecture controlled by a streamlined FastAPI application server layer coupled directly to an agile, index-optimized MongoDB Atlas cloud database collection layer.

* Asynchronous Coroutines Lifespan Setup: Database lifecycle states, persistent connection pools (maxPoolSize=50), and model memory storage initialization tasks execute natively inside an isolated asynccontextmanager lifecycle engine. This eliminates startup latency bottlenecks and securely isolates configuration tasks from live runtime execution loops.

* State Verification Audits and Data Layouts: The core analytical interface wraps MediaPipe face frame arrays, OpenCV quality verification scoring loops (using Laplacian variant metrics to detect blur and LAB-space **L**-channels to track exposure), deep learning model paths, and database operations into a unified transaction block.

* Localized Neural RAG Automation: Upon receiving an inferred condition from the model, its label is projected into a localized hash space vector. This vector runs a cosine similarity calculation against a structured medical text matrix to retrieve highly relevant clinical context. The extracted context block is fed into an execution pipeline template to generate three personalized, biometric-aligned wellness interventions for the user dashboard.

## 5. Step-by-Step Setup

### Step 1 — Clone / place files

```bash
mkdir backend && cd backend
# Place app.py, requirements.txt, .env.example here
```

### Step 2 — Create a virtual environment

```bash
python3 -m venv .venv

# Activate:
source .venv/bin/activate        # macOS / Linux
.venv\Scripts\activate           # Windows PowerShell
```

### Step 3 — Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

> **GPU users**: edit `requirements.txt` — uncomment the CUDA torch lines and
> comment out the CPU ones before running pip install.

### Step 4 — Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in **at minimum**:

```
MONGODB_URI=mongodb+srv://YOUR_USERNAME:YOUR_PASSWORD@YOUR_CLUSTER.mongodb.net/?retryWrites=true&w=majority
DATABASE_NAME=wellness_analytics
```

**How to get your MongoDB URI:**
1. Log in to [MongoDB Atlas](https://cloud.mongodb.com)
2. Click your cluster → **Connect**
3. Choose **Drivers** → Python → copy the connection string
4. Replace `<password>` with your actual Atlas password

> ⚠️ **Never commit `.env` to git.** It's already in `.gitignore`.

### Step 5 — (Optional) Add model checkpoints

If you have trained `.pth` files, place them in `checkpoints/`:

```
checkpoints/
├── body_classifier.pth
├── disease_classifier.pth
├── patch_autoencoder.pth
└── class_mapping.json     ← optional; overrides default class lists
```

If checkpoints are absent, models run with **random weights** (demo mode — outputs are still generated).

### Step 6 — Run the server

```bash
# Development (with auto-reload)
uvicorn app:app --reload --host 0.0.0.0 --port 8000
 

# OR use the __main__ block directly
python app.py
```

Server starts at: **http://localhost:8000**
